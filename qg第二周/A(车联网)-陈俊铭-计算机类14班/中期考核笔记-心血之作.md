- [x] #### ==对论文的理解==

我尝试总结了一下这篇论文到底在干什么，让我之后的思路更清晰

这篇论文研究的是**如何让网联自动驾驶车辆在不同通信网络结构下形成稳定的队列**，简单说，就是每辆车根据自己和邻居车辆（包括领航车）的位置、速度差异，实时调整自己的加减速和转向，最终让所有车的速度保持一致，并且保持期望的纵向和横向间距。

> 很有意思，就是保持总体的稳定，所以涉及到收敛性和，鲁棒性，图论

**实验与发现**：

通过数值仿真，对比了**两种通信网络拓扑**：

**情况一**：通信网络连接比较紧密，所有车都能直接或间接地获得信息。

**情况二**：部分通信链路失效，网络结构变得稀疏。

**论文的结论**：

在两种网络下，车辆最终都能形成队列，但**网络结构越简单（连接越少），车辆间的相互影响越小，收敛速度反而更快**。

同时，系统的鲁棒性（抗干扰能力）在不同网络下都能得到保证。

> 本质上就是 **PD 控制器**
>
> 位置差（P 控制）（比例项）可以保证车与车之间的间距，但是只用位置差（P 控制），车辆会靠近但容易超调，反复震荡。所以加上速度差（D 项）（微分项），产生阻尼，避免来回摆动，让车辆平稳地停到目标位置。

**核心控制协议 公式如下：**

![1774439234718](C:\Users\epson\AppData\Roaming\Typora\typora-user-images\1774439234718.png)

​     

*例：*

​     *纯位置反馈（P）：*

```
u = -Kp * (x2 - (x1 - 2))
```

​     *当车 2 离目标远时，加速度大，靠近快；但到达目标时速度可能*

​    *不为零，就会冲过头，然后来回震荡。*

​    *加入速度差（D）：*

```
u = -Kp * (x2 - (x1 - 2)) - Kd * (v2 - v1)
```

​    *当车 2 速度比领导快时，第二项产生负加速度，让它减速；慢时则加速。这样就能平滑地停在目标位置。*



这个公式由 **两个负反馈项** 组成，前面都有一个负号，表示“朝误差减小的方向加速”。

论文公式 (3) 的 β 和 γ 就是速度差增益参数，它决定了多智能体系统收敛的快慢。β 和 γ越大，信息流动越快，编队收敛越快。这里把位置和速度的系数分开了。但如果增益**太大**，系统会变得 **震荡甚至不稳定**——就像开车时看到前车减速你就猛踩刹车，结果车身会晃。所以论文里会选择合适的增益，平衡速度和稳定性。

邻居项，当相对距离大于期望值的时候，就会产生负加速度，拉近距离。领导项，相对距离大于期望值的时候，也是产生负的加速度，拉近距离。



这个公式本质上是 **PD 控制器（比例‑微分）** 的分布式版本：

- **比例项**（位置差）：负责消除位置误差，保证最终间距正确。
- **微分项**（速度差）：提供阻尼，防止震荡，让车辆平滑地接近目标。

特别的，如果只用位置差，车辆会来回摆动；加上速度差就能平稳停下。

**简要的矩阵推导说明：**

已经能够理解论文中单个车辆的控制公式，现在要将这些公式写成矩阵形式，本质上是把**所有车辆**的方程整合成一个紧凑的向量/矩阵方程。这样做的好处是能利用线性代数和图论工具来整体分析系统的稳定性。

在代码中，只需要构建矩阵 L*L* 和 K*K*，然后直接用矩阵乘法更新状态，而**不需要循环求和**。这种形式可以很容易地扩展到三维、非线性情况或包含时延的系统。
